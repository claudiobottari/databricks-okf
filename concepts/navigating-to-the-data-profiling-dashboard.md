# Navigating to the Data Profiling Dashboard

**Navigating to the Data Profiling Dashboard** refers to the methods for accessing the dashboard that is automatically created when a [data profile](/concepts/data-profile-databricks.md) is run on a table. This dashboard displays key metrics computed by the profile and is customizable like any standard dashboard in Databricks.

## Overview

When a profile runs, it creates a dashboard that visualizes key metrics. The visualizations included in the default dashboard configuration depend on the profile type (Snapshot, Timeseries, or InferenceLog), and the different metrics are organized into sections. The left side of the dashboard shows lists of the metrics and statistics included in the tables and charts. ^[data-profiling-dashboard-databricks-on-aws.md]

The dashboard has user-editable parameters for both the entire dashboard and for each chart, allowing you to customize the date range, data slices, models, and so on. You can also modify the charts shown or add new ones. ^[data-profiling-dashboard-databricks-on-aws.md]

## Access Methods

To view the data profiling dashboard, you must use the Databricks workspace from which [Data Profiling](/concepts/data-profiling.md) was enabled. There are two primary navigation paths: ^[data-profiling-dashboard-databricks-on-aws.md]

### Method 1: Via Catalog Explorer

1. In the left sidebar, click the **Catalog** icon to open the [Catalog Explorer](/concepts/catalog-explorer.md) UI.
2. Navigate to the primary table in the catalog directory.
3. Click the **Quality** tab on the table details page.
4. Click **View dashboard**.

![Access dashboard from Catalog UI.](https://docs.databricks.com/aws/en/assets/images/dashboard-in-catalog-20190888cd8183a3f7595ababeb3b02b.png)

^[data-profiling-dashboard-databricks-on-aws.md]

### Method 2: Via Dashboards Listing

1. In the left sidebar, click the **Dashboards** icon.
2. By default, the dashboard listing page shows dashboards that you have access to, sorted in reverse chronological order.
3. To filter the list, type any part of the dashboard name into the search box. You can also filter by last modified date or by owner.

![Access dashboard from Dashboards on sidebar.](https://docs.databricks.com/aws/en/assets/images/dashboard-list-267c17cf62ba6d46c9c46c9ccf6d6564.png)

^[data-profiling-dashboard-databricks-on-aws.md]

## Dashboard Limitations and Refresh Behavior

The dashboard is created in the user's account and is customizable and shareable like any dashboard. For general information about using and customizing dashboards, including adding new charts, editing charts, viewing queries, and so on, see the Dashboards documentation. ^[data-profiling-dashboard-databricks-on-aws.md]

### Refresh Requirements

The dashboard displays metrics that have been calculated by the profile. To refresh the values shown on the dashboard, you must: ^[data-profiling-dashboard-databricks-on-aws.md]

1. **Trigger a profile refresh** using the UI or the API, or set up a scheduled run. You cannot refresh the metrics from the dashboard itself.
2. **Click the Refresh button on the dashboard** to update the visualizations. When you click **Refresh** on the dashboard, it runs the queries over the metric tables that the dashboard uses to generate visualizations — it does not trigger profile calculations.

### Important Distinctions

The metric tables and the dashboard generated by a profile are updated separately. When you trigger a profile refresh, the metric tables are updated, but the dashboard is not automatically updated. To see the latest data: ^[data-profiling-dashboard-databricks-on-aws.md]

- Update the profile → Updates the underlying metric tables.
- Update the dashboard → Refreshes the visualizations from the metric tables.
