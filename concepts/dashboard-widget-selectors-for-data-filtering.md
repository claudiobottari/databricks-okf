# Dashboard Widget Selectors for Data Filtering

**Dashboard Widget Selectors for Data Filtering** refers to the interactive controls at the top of a [Data profiling dashboard](/concepts/data-profiling-dashboard.md) that allow users to dynamically filter and customize which data is displayed in the dashboard's charts and tables. These selectors enable drill-down analysis by date ranges, data slices, models, and other user-editable parameters.

## Overview

When a profile is run on a table in [Unity Catalog](/concepts/unity-catalog.md), it creates a dashboard that displays key metrics computed by the profile. The dashboard includes user-editable parameters for both the entire dashboard and for each individual chart. ^[data-profiling-dashboard-databricks-on-aws.md]

The left side of the dashboard shows lists of the metrics and statistics included in the tables and charts. At the top of the dashboard, widget selectors appear to control what data is included in the visualizations. ^[data-profiling-dashboard-databricks-on-aws.md]

## Selector Types by Profile Analysis

The specific widget selectors that appear depend on the type of profile analysis being performed:

- **Snapshot analysis** – The dashboard displays selectors appropriate for a single point-in-time view of the data. ^[data-profiling-dashboard-databricks-on-aws.md]
- **Timeseries analysis** – Different selectors appear, typically enabling users to choose date ranges, time granularities, or metric windows. ^[data-profiling-dashboard-databricks-on-aws.md]
- **InferenceLog analysis** – A distinct set of selectors appears to filter by model versions, inference timestamps, or prediction outcomes. ^[data-profiling-dashboard-databricks-on-aws.md]

The source material does not enumerate every selector for each type, but confirms that the dashboard adapts its filtering controls to the analysis mode.

## Customizing Filters

Users can adjust widget parameters to refine the data shown:

- **Dashboard‑level parameters** – Global filters that apply to all charts on the dashboard, such as a date range or a model name.
- **Chart‑level parameters** – Per‑chart filters that allow independent customization of individual visualizations.

These parameters are user‑editable, meaning analysts can interact with the dashboard without modifying the underlying profile configuration. ^[data-profiling-dashboard-databricks-on-aws.md]

## Relationship to Profile Refreshes

The widget selectors control only the data that is queried from the metric tables that the dashboard reads. They do **not** trigger a new profile calculation. To update the underlying metric tables, a profile refresh must be run first (via the UI or API). After the profile is refreshed, clicking the dashboard's **Refresh** button re‑executes the dashboard queries with the current selections and displays updated visualizations. ^[data-profiling-dashboard-databricks-on-aws.md]

## Best Practices

- Use the dashboard‑level selectors to quickly pivot between time periods or data segments.
- Apply chart‑level selectors when comparing different slices of data side‑by‑side.
- Remember that the selectors filter what is **displayed**; to get fresh data, refresh the profile, then refresh the dashboard.
- To create custom filters beyond the default widgets, modify the dashboard’s charts or add new ones using the dashboard editor.
