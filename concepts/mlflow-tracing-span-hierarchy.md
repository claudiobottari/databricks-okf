# [MLflow Tracing](/concepts/mlflow-tracing.md) Span Hierarchy

**MLflow Tracing Span Hierarchy** refers to the parent-child relationship between Span objects within a single [[MLflow Trace]]. A trace is a tree of spans, where each span represents a unit of work (e.g., an LLM call, a custom function, or a sub‑workflow). The hierarchy captures the order and nesting of operations, making it possible to understand execution bottlenecks and causal dependencies.

## Overview

When you instrument your GenAI application with [MLflow Tracing](/concepts/mlflow-tracing.md), every tracked operation becomes a span. Spans are organized in a tree: a parent span can contain one or more child spans, which themselves can have further children. The root span (the top‑most parent) represents the entire workflow. In the MLflow UI, this hierarchy is visualized as a nested timeline. ^[automatic-tracing-databricks-on-aws.md]

The hierarchy is created automatically by both [Automatic Tracing](/concepts/automatic-tracing.md) and [Manual Tracing](/concepts/manual-tracing.md) (via the `@mlflow.trace` decorator). Mixing the two approaches produces a single, unified trace with spans from both sources. ^[automatic-tracing-databricks-on-aws.md]

## Automatic Tracing

When autologging is enabled for a supported library (e.g., OpenAI, LangChain), MLflow automatically creates a child span for each API call made to that library. These spans are nested inside any parent span that you define manually with `@mlflow.trace`. ^[automatic-tracing-databricks-on-aws.md]

For example, calling `client.chat.completions.create()` inside a function decorated with `@mlflow.trace` produces a trace where the function span is the parent and the OpenAI call span is a child. ^[automatic-tracing-databricks-on-aws.md]

## Manual Tracing

The `@mlflow.trace` decorator — optionally specifying a SpanType such as `SpanType.CHAIN` — creates a manual span. When you call an auto‑traced library inside the decorated function, the auto‑traced span becomes a child of the manual span. This allows you to build arbitrarily deep hierarchies. ^[automatic-tracing-databricks-on-aws.md]

```python
import mlflow
from mlflow.entities import SpanType

mlflow.openai.autolog()

@mlflow.trace(span_type=SpanType.CHAIN)
def process_user_query(query: str):
    # First LLM call – automatically traced child span
    analysis = client.chat.completions.create(...)
    # Second LLM call – another child span
    response = client.chat.completions.create(...)
    return response.choices[0].message.content
```

Running this code creates a trace with one parent span (`process_user_query`) and two child spans (the two OpenAI calls). ^[automatic-tracing-databricks-on-aws.md]

## Multi‑Framework Traces

When automatic tracing is enabled for more than one framework (e.g., both OpenAI and LangChain), all API calls from any of those frameworks are captured as child spans under the same parent. This produces a unified hierarchy that spans multiple libraries. ^[automatic-tracing-databricks-on-aws.md]

## Example: Combined Manual and Automatic Tracing

The following snippet generates a single trace that combines manual spans with automatic OpenAI spans: ^[automatic-tracing-databricks-on-aws.md]

```python
@mlflow.trace(span_type=SpanType.CHAIN)
def run(question):
    messages = build_messages(question)
    # OpenAI call – automatically traced child span
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
    )
    return parse_response(response)

@mlflow.trace
def build_messages(question): ...

@mlflow.trace
def parse_response(response): ...
```

The resulting trace has a root span (`run`) with child spans for `build_messages`, the OpenAI call, and `parse_response`. The OpenAI call itself is a leaf span. ^[automatic-tracing-databricks-on-aws.md]

## Best Practices

- Use `SpanType.CHAIN` for high‑level workflow spans that contain multiple LLM calls. ^[automatic-tracing-databricks-on-aws.md]
- Keep manual spans coarse‑grained (one per logical step) to avoid excessive nesting. ^[automatic-tracing-databricks-on-aws.md]
- Enable autologging for all libraries used in your application so that every important API call is captured as a child span. ^[automatic-tracing-databricks-on-aws.md]
