## Accessing Data Profiling Dashboards in Databricks

**Accessing Data Profiling Dashboards in Databricks** describes the methods for locating and viewing the dashboard that is automatically created when a [Data Profiling](/concepts/data-profiling.md) monitor runs. This dashboard displays key metrics computed by the profile, with visualizations organized into sections based on the profile type (e.g., `Snapshot`, `TimeSeries`, `InferenceLog`). The left side of the dashboard shows lists of the metrics and statistics included in the tables and charts. The dashboard is created in the user's account and is customizable and shareable like any standard Databricks dashboard. ^[data-profiling-dashboard-databricks-on-aws.md]

### Viewing the Dashboard

To view the data profiling dashboard, you must use the Databricks workspace from which data profiling was enabled. There are two primary navigation paths: ^[data-profiling-dashboard-databricks-on-aws.md]

**Via the Catalog Explorer:**

1. In the left sidebar, click the **Catalog** icon to open the Catalog Explorer UI.
2. Navigate to the primary (monitored) table.
3. Click the **Quality** tab.
4. Click **View dashboard**.

**Via the Dashboards page:**

1. In the left sidebar, click the **Dashboards** icon.
2. By default, the dashboard listing page shows dashboards you have access to, sorted in reverse chronological order.
3. To filter the list, type any part of the dashboard name into the search box. You can also filter by last modified within a time period or by owner.

### Selecting Data to Display

Use the widgets at the top of the dashboard to control what data is included. For `Snapshot` analysis, the selectors differ from those shown for `TimeSeries` and `InferenceLog` analysis types. ^[data-profiling-dashboard-databricks-on-aws.md]

### Refreshing Dashboard Data

The dashboard displays metrics that have been calculated by the profile. The metric tables and the dashboard generated by a profile are updated separately. To update the data shown on the dashboard, you must follow a two-step process: ^[data-profiling-dashboard-databricks-on-aws.md]

1. **Trigger a profile refresh** using the UI or API to update the underlying [Profile Metrics Table](/concepts/profile-metrics-table.md) and [Drift Metrics Table](/concepts/drift-metrics-table.md).
2. **Click the Refresh button on the dashboard** to run the queries over the updated metric tables that the dashboard uses to generate visualizations.

You cannot refresh the profile metrics from the dashboard itself. Clicking **Refresh** on the dashboard does not trigger profile calculations; it only re-runs the existing queries. Modifying the dashboard charts or filters does not cause statistics to be recalculated. ^[data-profiling-dashboard-databricks-on-aws.md]

### Customization

The dashboard has user-editable parameters for both the entire dashboard and for each chart, allowing customization of the date range, data slices, models, and so on. You can also modify the existing charts shown or add new ones. For general guidance on using and customizing dashboards, including editing charts and viewing queries, see the Databricks documentation on Dashboards. ^[data-profiling-dashboard-databricks-on-aws.md]
