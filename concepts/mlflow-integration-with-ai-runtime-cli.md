# MLflow integration with AI Runtime CLI

The **AI Runtime CLI** (`air`) integrates with [MLflow Tracking](/concepts/mlflow-tracking.md) to automatically log experiment metadata, metrics, parameters, and artifacts for every workload submitted through the CLI. This integration provides visibility into training runs without requiring users to add explicit MLflow logging calls to their training code. ^[ai-runtime-cli-databricks-on-aws.md]

## Automatic MLflow Tracking

When you submit a training workload using the `air run` command, AI Runtime automatically creates an [MLflow Run](/concepts/mlflow-run.md) for the job. The following information is captured without any additional instrumentation in your training script: ^[ai-runtime-cli-databricks-on-aws.md]

- **Run metadata**: Job name, run ID, status, and timestamps
- **Parameters**: Configuration values from the workload YAML file, including compute settings and hyperparameters
- **Metrics**: Training metrics reported during execution (e.g., loss, accuracy)
- **Artifacts**: Model checkpoints, logs, and other output files generated by the training process

## Viewing Runs

### In the Databricks Workspace

Each AI Runtime CLI workload appears as a run in the [MLflow Experiment](/concepts/mlflow-experiment.md) associated with the workspace. You can view runs in the Databricks workspace UI under the **Experiments** section or from the **Jobs run page** for the specific workload. ^[ai-runtime-cli-databricks-on-aws.md]

### From the CLI

The `air list` command displays metadata for submitted workloads, including run IDs that link to MLflow runs. Use `air status` to check the current state of a workload and retrieve the [MLflow Run](/concepts/mlflow-run.md) URL for detailed tracking information. ^[ai-runtime-cli-databricks-on-aws.md]

## YAML Configuration for MLflow

The workload YAML file can include MLflow-related configuration under the `mlflow` section. This allows you to specify experiment names, tags, and other tracking settings declaratively as part of your job definition. ^[ai-runtime-cli-databricks-on-aws.md]

Example YAML snippet:

```yaml
name: my-training-job
mlflow:
  experiment_name: /Users/me/my-experiment
  tags:
    team: ml
    project: fine-tuning
```

^[ai-runtime-cli-databricks-on-aws.md]

## Model Registration

After a training run completes, you can register the resulting model in [Unity Catalog](/concepts/unity-catalog.md) or the [MLflow Model Registry](/concepts/mlflow-model-registry.md) directly from the CLI or through the Databricks workspace UI. This enables centralized model governance, versioning, and deployment as described in [AI Governance with Unity Catalog](/concepts/ai-governance-with-unity-catalog.md). ^[ai-runtime-cli-databricks-on-aws.md]

## Benefits of the Integration

- **No code changes required**: MLflow tracking works automatically without modifying training scripts
- **Reproducibility**: All run parameters and configuration are captured, making it easy to reproduce results
- **Comparison**: Multiple runs can be compared side-by-side in the MLflow UI to select the best model
- **Audit trail**: Every training run is logged with full provenance, supporting compliance and governance requirements
