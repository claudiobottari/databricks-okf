# Activation Link for Delta Sharing Recipients

The **Activation Link for Delta Sharing Recipients** is a URL generated by the data provider when creating a recipient object for [Delta Sharing](/concepts/delta-sharing.md) using the bearer token authentication method (also known as Databricks-to-Open sharing). The activation link allows a recipient who does not have access to a Unity Catalog-enabled Databricks workspace to securely download a credential file and access shared data. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

## How It Works

When a data provider creates a recipient object in their Unity Catalog [Metastore](/concepts/metastore.md) using the bearer token method, Databricks generates three components: a token, a credential file that includes the token, and an activation link. The provider shares the activation link with the recipient over a secure channel. When the recipient accesses the link, they download the credential file, which they then use to authenticate and obtain read access to the tables included in the shares granted to them. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

## Obtaining the Activation Link

The activation link is displayed immediately after creating a recipient in Catalog Explorer. It can also be retrieved later using Catalog Explorer, the Databricks Unity Catalog CLI, or the `DESCRIBE RECIPIENT` SQL command. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

**Required permissions**: [Metastore](/concepts/metastore.md) admin, user with the `USE RECIPIENT` privilege, or the recipient object owner. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

**Important**: If the recipient has already downloaded the credential file, the activation link is no longer returned or displayed. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

## Security Considerations

The activation link and the credential file it provides access to should be treated as secrets. Recipients must not share the credential outside of their organization. The credential file can only be downloaded once. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

If an activation link is inadvertently sent to the wrong person or sent over an insecure channel, Databricks recommends the following response:

1. Revoke the recipient's access to the share.
2. Rotate the recipient's token with `--existing-token-expire-in-seconds` set to `0`.
3. Share the new activation link with the intended recipient over a secure channel.
4. After the activation URL has been accessed, grant the recipient access to the share again.

^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

## Token Rotation and Activation Link Replacement

When a recipient's token is rotated, a new activation link is generated. The provider must share the new activation link with the recipient over a secure channel. This is necessary when:

- The existing recipient token is about to expire.
- The activation URL is lost or compromised.
- The credential is corrupted, lost, or compromised after download.
- The recipient token lifetime for the [Metastore](/concepts/metastore.md) has been modified.

^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]

If the existing activation URL has never been accessed, rotating the token invalidates that activation URL and replaces it with a new one. If all recipient tokens have expired, rotating the token also replaces the existing activation URL with a new one. ^[create-a-recipient-object-for-non-databricks-users-using-bearer-tokens-databricks-to-open-sharing-databricks-on-aws.md]
