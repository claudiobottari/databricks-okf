# Dashboard Access Methods

There are two primary methods to access the [Data profiling dashboard](/concepts/data-profiling-dashboard.md) from the workspace where data profiling was enabled. The dashboard is automatically created when a profile run completes and displays key metrics computed by the profile. ^[data-profiling-dashboard-databricks-on-aws.md]

## Access via Catalog Explorer

1. In the left sidebar, click the **Catalog** icon to open the [Catalog Explorer](/concepts/catalog-explorer.md) UI.
2. Navigate to the primary table on which the profile was run.
3. Click the **Quality** tab.
4. Click **View dashboard**.

The dashboard opens, displaying the metrics from the most recent profile run. ^[data-profiling-dashboard-databricks-on-aws.md]

## Access via the Dashboards List

1. In the left sidebar, click the **Dashboards** icon.
2. The dashboard listing page shows all dashboards you have access to, sorted in reverse chronological order by default.
3. To find the data profiling dashboard:
   - Type any part of the dashboard name into the search box.
   - Filter by **Last modified** within a time period, or by **Owner**.

The dashboard name typically includes the table name or a related identifier. Once located, click it to open. ^[data-profiling-dashboard-databricks-on-aws.md]

## Refreshing Dashboard Data

The dashboard displays metrics that have been calculated by the profile. To update the values shown:

1. First trigger a profile refresh using [the UI](https://docs.databricks.com/aws/en/data-governance/unity-catalog/data-quality-monitoring/data-profiling/create-monitor-ui#refresh) or [the API](https://docs.databricks.com/aws/en/data-governance/unity-catalog/data-quality-monitoring/data-profiling/create-monitor-api#refresh), or set up a [scheduled profile run](/concepts/profile-refresh-and-scheduling.md).
2. Then click the **Refresh** button on the dashboard to re-run the queries over the updated metric tables.

The metric tables and the dashboard generated by a profile are updated separately. When you trigger a profile refresh, the metric tables are updated, but the dashboard is not automatically updated. Refreshing the dashboard alone does not trigger profile recalculations; it only updates the visualizations from the existing metric tables. ^[data-profiling-dashboard-databricks-on-aws.md]
