# Delta Lake Manifest Files for External Query Engines

**Delta Lake Manifest Files** are files generated by the `GENERATE` command that allow external query engines—such as Presto and Amazon Athena—to read [Delta Lake](/concepts/delta-lake.md) tables without direct Delta Lake integration. The manifest files are produced in a **symlink format** that these engines can interpret as a directory of data files.^[generate-databricks-on-aws.md]

## Generating Manifest Files

To create a manifest file for a Delta table, use the `GENERATE` command with the `symlink_format_manifest` mode:

```sql
GENERATE symlink_format_manifest FOR TABLE table_name
```

- `symlink_format_manifest` is the only mode parameter supported for this command. It instructs Delta Lake to write one or more manifest files that contain references to the underlying Parquet (or other) data files composing the table.^[generate-databricks-on-aws.md]
- `table_name` must identify an existing Delta table. The name cannot include a temporal specification or options specification.^[generate-databricks-on-aws.md]

After generation, external query engines like Presto and Athena can read the manifest file location as if it were a table, enabling them to query Delta data without native Delta Lake support.^[generate-databricks-on-aws.md]

## Use Cases

- **Cross-engine compatibility**: Enable query engines that do not support Delta Lake directly (e.g., Presto, Athena) to read Delta tables.
- **Batch analytics**: Allow ETL pipelines that rely on Presto or Athena to consume Delta Lake tables.
- **Migration scenarios**: Simplify the transition of existing Presto/Athena workflows to Delta Lake by providing a compatible manifest interface.
