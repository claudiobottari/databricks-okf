# User and Session Tracking in GenAI Traces

**User and Session Tracking in GenAI Traces** refers to the practice of attaching user identifiers and session metadata to execution traces generated by GenAI applications. This enables developers to analyze user behavior, track multi-turn conversations, personalize experiences, and monitor application quality across different user segments.

## Overview

Modern GenAI applications often serve multiple users simultaneously, with each user engaging in multi-turn conversations that span multiple interactions. Tracking users and sessions provides essential context for understanding how different users interact with the application, analyzing conversation flows, and maintaining context across multiple interactions.^[add-context-to-traces-databricks-on-aws.md]

## Standard Metadata Fields

MLflow provides two standardized metadata fields for user and session tracking in traces:

- **`mlflow.trace.user`** — Associates traces with specific users, enabling per-user analysis and personalization insights.
- **`mlflow.trace.session`** — Groups traces belonging to multi-turn conversations, enabling conversation flow tracking and session continuity analysis.

These fields are accessible via `metadata` in the pandas DataFrame returned by `mlflow.search_traces()`, or by using the `Trace.info.trace_metadata` field from `Trace` objects. When using these standard fields, MLflow automatically enables filtering and grouping in the UI.^[add-context-to-traces-databricks-on-aws.md]

## Implementation

To add user and session tracking to traces, you call `mlflow.update_current_trace()` during your application's execution, attaching metadata to the active trace:

```python
import mlflow

mlflow.update_current_trace(
    metadata={
        "mlflow.trace.user": user_id,
        "mlflow.trace.session": session_id,
    },
)
```

The most common approach is to use the `@mlflow.trace` decorator to trace functions automatically, then call `update_current_trace()` within the traced function to associate the current user and session.^[add-context-to-traces-databricks-on-aws.md]

## Why Track Users and Sessions?

User and session tracking enables several powerful analytics capabilities:

1. **User behavior analysis** — Understand how different users interact with your application.
2. **Conversation flow tracking** — Analyze multi-turn conversations and context retention.
3. **Personalization insights** — Identify patterns to improve user-specific experiences.
4. **Quality per user** — Track performance metrics across different user segments.
5. **Session continuity** — Maintain context across multiple interactions.

## Accessing Tracked Data

Once traces are logged with user and session metadata, you can access them programmatically:

- Use `mlflow.search_traces()` to query traces, filtering by user or session.
- Access `Trace.info.trace_metadata` to retrieve the stored metadata dictionary.
- Access `Trace.info.tags` for mutable tag data.
- The `metadata` and `tags` fields are also available in the pandas DataFrame returned by `mlflow.search_traces()`.

## Best Practices

1. **Consistent ID formats** — Use standardized formats for user and session IDs across your application.^[add-context-to-traces-databricks-on-aws.md]
2. **Session boundaries** — Define clear rules for when sessions start and end.^[add-context-to-traces-databricks-on-aws.md]
3. **Combine context types** — Track user, session, and environment context together for complete traceability.^[add-context-to-traces-databricks-on-aws.md]
4. **Regular analysis** — Set up dashboards to monitor user behavior, session patterns, and version performance.^[add-context-to-traces-databricks-on-aws.md]
5. **Override defaults thoughtfully** — Only override automatically populated metadata when necessary.^[add-context-to-traces-databricks-on-aws.md]

## Metadata Immutability

Unlike tags, metadata fields are immutable in the logged trace. Once a trace is logged, its metadata values cannot be changed. This makes metadata ideal for storing immutable identifiers like user and session IDs, while tags are better suited for values that may need updating after trace completion.^[add-context-to-traces-databricks-on-aws.md]

## Requirements

To use context tracking including user and session metadata, install the `mlflow-tracing` package for production environments. MLflow 3 is required; MLflow 2.x is not supported due to performance limitations and missing features essential for production use.^[add-context-to-traces-databricks-on-aws.md]
