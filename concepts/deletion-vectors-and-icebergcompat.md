# Deletion Vectors and IcebergCompat

**Deletion Vectors and IcebergCompat** describes the compatibility requirements between [Deletion Vectors](/concepts/deletion-vectors.md) and Databricks’ IcebergCompat table feature. When enabling IcebergCompat on a table, Deletion Vectors must be disabled and fully purged before the operation can succeed. The Delta Lake engine enforces this rule through two distinct error conditions. ^[delta_iceberg_compat_violation-error-condition-databricks-on-aws.md]

## Overview

IcebergCompatV`<version>` (currently versions 1 and 2) requires that Deletion Vectors are not present on the table. Deletion Vectors are a Delta Lake feature used to track soft-deleted rows without rewriting data files, but their presence is incompatible with the uniform Apache Iceberg format produced by IcebergCompat. Therefore, a table that has Deletion Vectors enabled must first disable them and then purge any remaining deletion vectors before IcebergCompat can be turned on. ^[delta_iceberg_compat_violation-error-condition-databricks-on-aws.md]

## DELETION_VECTORS_NOT_PURGED

This error occurs when Deletion Vectors have already been disabled on the table, but some remaining deletion vector metadata or artifacts still exist. The error message is:

```
IcebergCompatV<version> requires Deletion Vectors to be completely purged from the table.
```

**Resolution:** Run the `REORG TABLE APPLY (PURGE)` command to remove all residual deletion vectors from the table. ^[delta_iceberg_compat_violation-error-condition-databricks-on-aws.md]

## DELETION_VECTORS_SHOULD_BE_DISABLED

This error occurs when Deletion Vectors are still actively enabled on the table. The error message is:

```
IcebergCompatV<version> requires Deletion Vectors to be disabled on the table first. Then run `REORG PURGE` command to purge the Deletion Vectors on the table.
```

**Resolution:** First disable Deletion Vectors on the table (for example, by setting the appropriate table property). Then run `REORG TABLE APPLY (PURGE)` to purge any remaining deletion vectors. After both steps, IcebergCompat can be enabled. ^[delta_iceberg_compat_violation-error-condition-databricks-on-aws.md]

## Resolution Steps

The general workflow to resolve both errors is:

1. **Disable Deletion Vectors** on the table if not already disabled.
2. **Purge** remaining deletion vectors using:
   ```
   REORG TABLE APPLY (PURGE)
   ```
3. After purging, enable IcebergCompat by running:
   ```
   REORG TABLE APPLY (UPGRADE UNIFORM ('ICEBERG_COMPAT_VERSION = <version>'))
   ```

Note that `REORG` enables the IcebergCompat table feature. Some Databricks runtime versions without support for that feature may not be able to write to the table afterward. ^[delta_iceberg_compat_violation-error-condition-databricks-on-aws.md]
