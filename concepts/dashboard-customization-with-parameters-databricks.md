# Dashboard Customization with Parameters (Databricks)

**Dashboard Customization with Parameters** refers to the ability to modify the appearance and data scope of dashboards generated by [Data Profiling](/concepts/data-profiling.md) in [Unity Catalog](/concepts/unity-catalog.md). These dashboards are produced automatically when a profile is run and include built-in, user-editable parameters that allow filtering by date range, data slices, models, and other dimensions. ^[data-profiling-dashboard-databricks-on-aws.md]

## Overview

When a data profile executes, it creates a dashboard that displays key metrics computed by the profile. The dashboard is stored in the user's account and is fully customizable and shareable. The left side of the dashboard lists the metrics and statistics included in the tables and charts. ^[data-profiling-dashboard-databricks-on-aws.md]

## Parameters

Both the entire dashboard and individual charts support user-editable parameters. These parameters control:

- **Date range** – Select a time window for the displayed data.
- **Data slices** – Filter to a subset of the profiled data (e.g., by category or region).
- **Models** – Choose which model or version to analyze (for inference‑log profiles).
- Other custom dimensions exposed by the underlying profile.

The exact set of available selectors depends on the profile type (`Snapshot`, `Timeseries`, or `InferenceLog`). ^[data-profiling-dashboard-databricks-on-aws.md]

## Customization

Users can modify the dashboard beyond parameter selection:

- **Edit or remove existing charts** – Change visualization types, queries, or layout.
- **Add new charts** – Create additional visualizations based on the profile’s metric tables.
- **Share the dashboard** – Grant access to other workspace users.

For general guidance on editing dashboards, adding charts, viewing queries, and sharing, see Dashboards documentation. ^[data-profiling-dashboard-databricks-on-aws.md]

## Important Notes

- Dashboard metrics are not recalculated when you modify the dashboard. To update the underlying data, you must trigger a profile refresh (via the UI or API) and then click the **Refresh** button on the dashboard to re‑run the dashboard queries over the updated metric tables. ^[data-profiling-dashboard-databricks-on-aws.md]
- The dashboard’s visualizations only reflect data that has already been computed by a profile run. They do not trigger new profile calculations. ^[data-profiling-dashboard-databricks-on-aws.md]
