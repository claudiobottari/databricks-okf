# system.data_classification.results System Table

The **`system.data_classification.results`** system table stores the output of [Data Classification](/concepts/data-classification.md) scanning across all catalogs in a [Unity Catalog](/concepts/unity-catalog.md) [Metastore](/concepts/metastore.md). It is the authoritative record of detected sensitive-data classifications, including sample values from classified columns. ^[data-classification-databricks-on-aws.md]

## Contents

The table contains all classification results generated by the Data Classification engine. For each detection it records:

- The classification tag (e.g., `class.email_address`, `class.ssn`).
- The column where the tag was detected.
- Sample values from that column.
- The `catalog_id` of the catalog that owns the table, which can be used to link with billing data in `system.billing.usage`. ^[data-classification-databricks-on-aws.md]

Because the table is metastore-wide, it includes detections from every catalog that has data classification enabled. ^[data-classification-databricks-on-aws.md]

## Access and Security

By default, only the account admin has access to this table. The account admin can grant `SELECT` permission to other users or groups. The table is only queryable when using serverless compute. ^[data-classification-databricks-on-aws.md]

> **Important**: The table contains sample values from tables across all catalogs. Access should be limited to users who are authorised to see metastore-wide classification results, including the sample values. ^[data-classification-databricks-on-aws.md]

Users who have `SELECT` on this table can also view sample values in the **Data Classification** results page in the UI, even without individual table access. ^[data-classification-databricks-on-aws.md]

## Usage

### Viewing classification results in the UI

When a user with `SELECT` on the table views the **Data Classification** results page, sample values from detected columns are displayed. Without `SELECT` on the table, sample values are hidden. ^[data-classification-databricks-on-aws.md]

### Joining with billing data

The `catalog_id` column in this table can be used to break down [Data Classification](/concepts/data-classification.md) costs by catalog when querying `system.billing.usage`. For example:

```sql
SELECT
  u.usage_date,
  u.usage_metadata.catalog_id,
  SUM(u.usage_quantity) AS dbus
FROM system.billing.usage AS u
WHERE u.billing_origin_product = 'DATA_CLASSIFICATION'
  AND u.usage_date >= DATE_SUB(CURRENT_DATE(), 30)
GROUP BY usage_date, catalog_id
ORDER BY usage_date DESC;
```
