# AssessmentSource Entity for Feedback Attribution

The **AssessmentSource** entity is an MLflow data model used to attribute feedback or evaluations to a specific origin. When logging user feedback or automated assessments against a trace, the `AssessmentSource` object captures who or what provided the feedback, enabling traceability and auditability in production monitoring workflows.

## Fields

The `AssessmentSource` entity has two fields:

| Field | Type | Description |
|-------|------|-------------|
| `source_type` | string | The category of the source. Common values are `"HUMAN"` for end‑user feedback and `"LLM_JUDGE"` for automated evaluation by a language model judge. |
| `source_id` | string | A unique identifier for the specific user, system, or service that provided the feedback. For human users this could be a user ID; for automated judges it might be a scorer name or endpoint identifier. |

## Usage in Feedback Logging

The `AssessmentSource` object is passed as the `source` parameter to the `mlflow.log_feedback()` API. It is the mechanism by which each piece of feedback is linked to its provider, whether human or automated. ^[collect-user-feedback-databricks-on-aws.md]

### Example: Human Feedback

In a typical production setup, when an end user submits a thumbs‑up/thumbs‑down, the feedback handler creates an `AssessmentSource` with `source_type="HUMAN"` and `source_id` set to the user’s identifier:

```python
from mlflow.entities import AssessmentSource

mlflow.log_feedback(
    trace_id=trace_id,
    name="user_feedback",
    value=is_correct,
    source=AssessmentSource(
        source_type="HUMAN",
        source_id=user_id
    ),
    rationale=comment
)
```

### Example: Multi‑Dimensional Ratings

The same pattern applies when logging multiple rating dimensions (e.g., accuracy, helpfulness, relevance) for a single trace. Each dimension is logged as a separate assessment, all referencing the same `AssessmentSource`:

```python
dimensions = {"accuracy": accuracy, "helpfulness": helpfulness, "relevance": relevance}
for dimension, score in dimensions.items():
    mlflow.log_feedback(
        trace_id=trace_id,
        name=f"user_{dimension}",
        value=score / 5.0,
        source=AssessmentSource(
            source_type="HUMAN",
            source_id=user_id
        ),
        rationale=comment if dimension == "accuracy" else None
    )
```

## Source Types

The `source_type` field is flexible but the documented standard values are:

- **`HUMAN`** – Used when the feedback originates from a person, such as an end‑user rating an AI response or an annotator providing labels.
- **`LLM_JUDGE`** – Used when the feedback is generated by an automated evaluation model (e.g., a custom LLM scorer registered via the [Code-based Scorers](/concepts/code-based-scorers.md) framework).

Using a consistent `source_type` vocabulary allows downstream analysis to distinguish human‑provided ground truth from automated assessments. ^[collect-user-feedback-databricks-on-aws.md]
