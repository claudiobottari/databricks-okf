# Databricks Connect Logging and Debug Logs

**Databricks Connect Logging and Debug Logs** refers to the diagnostic output generated by [[Databricks Connect]] during client-side Spark operations. Logging is essential for troubleshooting connection issues, understanding gRPC communication, and monitoring client-side behavior.

## Overview

Databricks Connect for Python produces logs using the standard Python logging module. Logs are emitted to the standard error stream (*stderr*). By default, logging is turned off — no log messages are printed unless explicitly enabled. ^[advanced-usage-of-databricks-connect-databricks-on-aws.md]

## Enabling Debug Logs

To enable logging, set the environment variable `SPARK_CONNECT_LOG_LEVEL` to `debug`. This modification activates all log messages at the `DEBUG` level and higher, making the full range of diagnostic information available. ^[advanced-usage-of-databricks-connect-databricks-on-aws.md]

A typical way to set the variable before creating a session is:

```python
import os
os.environ["SPARK_CONNECT_LOG_LEVEL"] = "debug"
```

After setting the environment variable (before initializing the `DatabricksSession`), log entries will appear in stderr as the client communicates with the Databricks cluster.

## Log Level Control

- **Default:** Logging is disabled; no messages are emitted to stderr.
- **`debug`:** Enables all `DEBUG`-level logs and higher (e.g., `INFO`, `WARNING`, `ERROR`).

## Scala Client

For Databricks Connect with Scala, the source material does not provide specific details on logging configuration. Users should refer to the Databricks documentation for Scala-specific logging options.
