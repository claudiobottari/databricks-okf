## AutoML import_notebook()

The `databricks.automl.import_notebook` method imports a notebook that has been saved as an MLflow artifact. It returns an AutoML Python API reference#ImportNotebookResult|ImportNotebookResult object. ^[automl-python-api-reference-databricks-on-aws.md]

### Signature

Python

    databricks.automl.import_notebook(  artifact_uri: str,  path: str,  overwrite: bool = False) -> ImportNotebookResult:

^[automl-python-api-reference-databricks-on-aws.md]

### Parameters

- **artifact_uri** (str): The URI of the MLflow artifact containing the notebook to import.
- **path** (str): The destination path in the Databricks workspace where the notebook will be placed.
- **overwrite** (bool, optional, default `False`): Whether to overwrite an existing notebook at the destination path. If `False` and a notebook already exists at the path, the import fails.

^[automl-python-api-reference-databricks-on-aws.md]

### Return value

Returns an `ImportNotebookResult` object. The result object has two attributes:

- **path** (str): The path where the notebook was imported.
- **url** (str): A URL that can be used to open the imported notebook.

^[automl-python-api-reference-databricks-on-aws.md]

### Example

The following example shows how to call `import_notebook` after running an `automl.classify` run. It imports the trial notebook from the fifth trial (index 5) into a specific directory in the workspace:

Python

    summary = databricks.automl.classify(...)result = databricks.automl.import_notebook(summary.trials[5].artifact_uri, "/Users/you@yourcompany.com/path/to/directory")print(result.path)print(result.url)

^[automl-python-api-reference-databricks-on-aws.md]

The example uses `summary.trials[5].artifact_uri` to get the artifact URI of the fifth trial, then passes it to `import_notebook` to place the notebook at `/Users/you@yourcompany.com/path/to/directory`. It prints the resulting path and URL.

### Use case

This method is useful when you want to bring a trial notebook generated by an AutoML run—which contains the model training code—into a specific folder in your Databricks workspace for further editing or inspection. The imported notebook is a standard Databricks notebook that you can open, modify, and run.
