# Trace Input/Output Tracking

**Trace Input/Output Tracking** is a core capability of [MLflow Tracing](/concepts/mlflow-tracing.md) that captures the complete request-response cycle of an application, including inputs, outputs, and metadata for each intermediate step such as retrieval, tool calls, and LLM interactions. This provides developers with deep visibility into application behavior across both development and production environments. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

## Overview

Trace Input/Output Tracking records the full execution flow of a GenAI application, enabling developers to visualize and understand the application's logic and decision-making process. By examining the inputs, outputs, and metadata for each intermediate step, teams can debug issues, monitor performance, and optimize costs. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

## Use Cases

### Development Debugging

In development environments, Trace Input/Output Tracking provides detailed visibility into what happens beneath the abstractions of GenAI libraries. This helps developers precisely identify where issues or unexpected behaviors occur, facilitating rapid iteration and debugging. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

### Production Monitoring

In production, traces capture errors and can include operational metrics like latency at each step, aiding in quick diagnostics. This allows teams to monitor and debug issues in real-time as they affect end users. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

## Key Features

- **Unified experience**: [MLflow Tracing](/concepts/mlflow-tracing.md) offers a consistent experience between development and production—you instrument your application once, and tracing works in both environments. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]
- **Seamless navigation**: You can navigate traces within your preferred environment—IDE, notebook, or production monitoring dashboard—eliminating the need to switch between multiple tools or search through overwhelming logs. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]
- **OpenTelemetry compatibility**: [MLflow Tracing](/concepts/mlflow-tracing.md) is compatible with OpenTelemetry, allowing you to export trace data to various services in your existing observability stack. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

## Performance and Cost Optimization

Trace Input/Output Tracking enables capturing and monitoring key operational metrics such as latency, cost, and resource utilization at each step of application execution. This allows teams to:

- Track and identify performance bottlenecks within complex pipelines
- Monitor resource utilization to ensure efficient operation
- Optimize cost efficiency by understanding where resources or tokens are consumed
- Identify areas for performance improvement in code or model interactions

^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

## Trace Analysis with Genie Code

Genie Code provides a natural language interface for exploring and debugging traces. Instead of writing queries or navigating multiple UI pages, you can ask questions like "Are there any error traces in this experiment?" or "What's the P95 latency for my traces?" and get immediate answers.

Genie Code has read access to traces, sessions, evaluation runs, scorers, datasets, labeling sessions, and more—enabling users to go from a high-level question to root cause analysis in a single conversation. ^[debug-and-analyze-your-app-with-tracing-databricks-on-aws.md]

## Data Captured

For each trace, the system captures:

- **Inputs**: The incoming request data passed to the application
- **Outputs**: The final response generated by the application
- **Intermediate steps**: Inputs, outputs, and metadata for each step in the execution flow (retrieval, tool calls, LLM interactions, etc.)
- **Operational metrics**: Latency, cost, and resource utilization at each step
- **Errors**: Any errors that occur during execution
