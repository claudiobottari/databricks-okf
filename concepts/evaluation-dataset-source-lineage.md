# Evaluation Dataset Source Lineage

**Evaluation Dataset Source Lineage** refers to the provenance tracking mechanism built into MLflow evaluation datasets. Each record in an evaluation dataset can carry a `source` field that documents how and from where that record was created, enabling teams to audit, reproduce, and understand the origin of test data used in [GenAI](/concepts/mlflow-genai-evaluate-api.md) evaluations. ^[evaluation-dataset-reference-databricks-on-aws.md]

## Source Field Schema

The `source` field is part of the evaluation dataset schema. It is optional but, when present, provides structured metadata about the record’s origin. A given record can have **only one** source type. ^[evaluation-dataset-reference-databricks-on-aws.md]

### Human Source

Indicates the record was created manually by a person. It includes the identifier of the user who created the record.

```python
{
    "source": {
        "human": {
            "user_name": "jane.doe@company.com"
        }
    }
}
```

^[evaluation-dataset-reference-databricks-on-aws.md]

### Document Source

Indicates the record was synthesized from a document (for example, a PDF, help article, or manual). It captures the document’s URI and can optionally include an excerpt of content.

```python
{
    "source": {
        "document": {
            "doc_uri": "s3://bucket/docs/product-manual.pdf",
            "content": "The first 500 chars of the document..."
        }
    }
}
```

^[evaluation-dataset-reference-databricks-on-aws.md]

### Trace Source

Indicates the record was created from a production trace — that is, a real invocation of your GenAI application recorded during [Production Monitoring](/concepts/production-monitoring.md). It stores the unique identifier of that trace.

```python
{
    "source": {
        "trace": {
            "trace_id": "tr-abc123def456"
        }
    }
}
```

^[evaluation-dataset-reference-databricks-on-aws.md]

## Viewing Source Trace in the UI

When a record originates from a production trace, the evaluation dataset UI displays the trace in the **Source** column. Clicking a trace identifier opens an interactive window that shows the full trace and its associated assessments, making it easy to inspect the exact production request that generated the record. ^[evaluation-dataset-reference-databricks-on-aws.md]

## Usage and Benefits

Source lineage makes it possible to:

- **Audit** how each test case was created — manually, from a document, or from production data.
- **Reproduce** the conditions under which a record was generated by referring to the original document or trace.
- **Monitor drift** over time by tracing evaluation datasets back to specific production traces or document versions.
- **Comply** with data governance requirements by maintaining a clear chain of custody for evaluation data.

While the source field is optional, recording it is a best practice for any serious [GenAI evaluation](/concepts/mlflow-genai-evaluation.md) workflow. ^[evaluation-dataset-reference-databricks-on-aws.md]
