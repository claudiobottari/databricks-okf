# Profile Refresh Mechanism

The **Profile refresh mechanism** refers to the process of updating the metrics and visualizations generated by [Data Profiling](/concepts/data-profiling.md) in Unity Catalog. Refreshing a profile involves two separate steps: updating the underlying metric tables and updating the dashboard that displays those metrics.

## Overview

When a profile runs, it computes key metrics and creates a dashboard that displays those metrics in visualizations. The metric tables and the dashboard are updated independently. To see current data on the dashboard, both the profile and the dashboard must be refreshed in sequence. ^[data-profiling-dashboard-databricks-on-aws.md]

## Refreshing the Profile

To update the metric tables that store the computed profile statistics, you must trigger a profile refresh. This can be done through:

- **The UI**: Navigate to the primary table in Catalog Explorer, open the **Quality** tab, and use the refresh option. ^[data-profiling-dashboard-databricks-on-aws.md]
- **The API**: Use the appropriate API endpoint to trigger a profile refresh programmatically. ^[data-profiling-dashboard-databricks-on-aws.md]
- **Scheduled runs**: Set up a recurring schedule for profile refreshes, configurable through either the UI or the API. ^[data-profiling-dashboard-databricks-on-aws.md]

You cannot refresh the profile metrics from the dashboard itself. ^[data-profiling-dashboard-databricks-on-aws.md]

## Refreshing the Dashboard

The dashboard displays metrics that have already been calculated by the profile. When you modify the dashboard, statistics are not recalculated. ^[data-profiling-dashboard-databricks-on-aws.md]

To update the data shown on the dashboard, click the **Refresh** button on the dashboard. This action runs the queries over the metric tables that the dashboard uses to generate visualizations. It does not trigger profile calculations. ^[data-profiling-dashboard-databricks-on-aws.md]

## Refresh Sequence

The correct sequence to update both the metrics and the dashboard is:

1. **Refresh the profile** to update the underlying metric tables with new computed statistics.
2. **Refresh the dashboard** to run queries against the updated metric tables and display the latest visualizations.

If you only refresh the dashboard without first refreshing the profile, the dashboard will show the same data as before. If you only refresh the profile without refreshing the dashboard, the dashboard will continue to display the old data. ^[data-profiling-dashboard-databricks-on-aws.md]
