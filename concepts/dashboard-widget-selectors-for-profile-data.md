# Dashboard Widget Selectors for Profile Data

**Dashboard Widget Selectors for Profile Data** are user-editable parameters at the top of a [Data profiling dashboard](/concepts/data-profiling-dashboard.md) that control which data is displayed in the dashboard's visualizations. These selectors allow you to customize the date range, data slices, models, and other aspects of the displayed metrics without recalculating the underlying profile. ^[data-profiling-dashboard-databricks-on-aws.md]

## Overview

When a profile runs, it creates a dashboard that displays key metrics computed by the profile. The dashboard includes user-editable parameters for both the entire dashboard and for each chart, allowing you to customize what data is shown. You can also modify the charts shown or add new ones. ^[data-profiling-dashboard-databricks-on-aws.md]

The selectors that appear depend on the profile type. For `Snapshot` analysis, different selectors appear compared to `Timeseries` and `InferenceLog` analysis. ^[data-profiling-dashboard-databricks-on-aws.md]

## Selector Types by Profile Analysis

### Snapshot Analysis

For `Snapshot` analysis profiles, the dashboard displays specific filters that control which snapshot data is included in the visualizations. ^[data-profiling-dashboard-databricks-on-aws.md]

### Timeseries Analysis

For `Timeseries` analysis profiles, different selectors appear to control the time-based data displayed. ^[data-profiling-dashboard-databricks-on-aws.md]

### InferenceLog Analysis

For `InferenceLog` analysis profiles, yet another set of selectors appears to control the inference log data shown. ^[data-profiling-dashboard-databricks-on-aws.md]

## Using Selectors

The selectors are located at the top of the dashboard. The left side of the dashboard also shows lists of the metrics and statistics included in the tables and charts. ^[data-profiling-dashboard-databricks-on-aws.md]

## Relationship to Profile Refreshes

It is important to understand that the selectors control which data is displayed from the metric tables, but they do not trigger new profile calculations. To update the data in the tables used to create the visualizations, you must refresh the profile using the UI or the API, or set up a scheduled run. After refreshing the profile, you must then click the **Refresh** button on the dashboard to update the data shown. ^[data-profiling-dashboard-databricks-on-aws.md]

The metric tables and the dashboard generated by a profile are updated separately. When you trigger a profile refresh, the metric tables are updated, but the dashboard is not automatically updated. Similarly, clicking **Refresh** on the dashboard does not trigger profile calculations — it runs the queries over the metric tables that the dashboard uses to generate visualizations. ^[data-profiling-dashboard-databricks-on-aws.md]
