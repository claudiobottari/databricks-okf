# Best practices for LLM judge instructions

**Best practices for LLM judge instructions** refers to the guidelines for writing effective natural language prompts that define how [Custom Judges](/concepts/custom-judges.md) should evaluate the quality of GenAI agent outputs.

## Overview

Custom LLM judges allow you to define nuanced scoring guidelines for GenAI applications using natural language. While MLflow provides [Built-in LLM Judges](/concepts/built-in-llm-judges.md) for common quality dimensions, custom judges created using make_judge()|Make Judge API|`make_judge()` give you full control over evaluation criteria. The quality of judge instructions directly impacts the accuracy and consistency of evaluations. ^[custom-judges-databricks-on-aws.md]

## Be Specific About Expected Output Format

Your instructions should clearly specify what format the judge should return. Different evaluation tasks require different response formats: ^[custom-judges-databricks-on-aws.md]

- **Categorical responses**: List specific values the judge can return (for example, `'fully_resolved'`, `'partially_resolved'`, `'needs_follow_up'`).
- **Boolean responses**: Explicitly state the judge should return `true` or `false`.
- **Numeric scores**: Specify the scoring range and what each score means.

## Break Down Complex Evaluations

For complex evaluation tasks, structure your instructions into clear sections. This helps the judge understand the evaluation process step by step: ^[custom-judges-databricks-on-aws.md]

- **What to evaluate** — Identify the specific quality dimension or behavior being assessed.
- **What information to examine** — Specify which template variables ({{ inputs }}, {{ outputs }}, {{ expectations }}, {{ trace }}) to analyze.
- **How to make the judgment** — Describe the reasoning process or criteria for scoring.
- **What format to return** — State the exact output structure expected.

## Use Appropriate Template Variables

Your instructions must include at least one template variable to access agent data. The allowed variables are: ^[custom-judges-databricks-on-aws.md]

- `{{ inputs }}` — Input data provided to the agent
- `{{ outputs }}` — Output data generated by your agent
- `{{ expectations }}` — Ground truths or expected outcomes
- `{{ trace }}` — The complete execution trace of your agent

Custom variables like `{{ question }}` will throw validation errors to ensure consistent behavior and prevent template injection issues. ^[custom-judges-databricks-on-aws.md]

## Align Judges with Human Experts

The base judge is a starting point. As you gather expert feedback on your application's outputs, you can align the LLM judges to the feedback to further improve judge accuracy. See [Align judges with humans](/concepts/aligning-judges-with-human-experts.md). ^[custom-judges-databricks-on-aws.md]
