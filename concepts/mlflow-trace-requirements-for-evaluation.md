## MLflow Trace Requirements for Evaluation

**MLflow Trace Requirements for Evaluation** define the structure and content that an [[MLflow Trace]] must satisfy in order for built-in [LLM Judge|LLM judges](/concepts/llm-judges.md) — such as `RelevanceToQuery` and `RetrievalRelevance` — to compute evaluation scores. These requirements ensure that the judge can locate the necessary data (inputs, outputs, or retrieved documents) within the trace.

### Overview

When using `mlflow.genai.evaluate()` or invoking a judge directly on a single trace, the trace must include specific span information. The exact requirements differ by judge and by how the trace was generated. All built-in judges rely on the trace's span structure to identify the data that should be scored. ^[answer-and-context-relevance-judges-databricks-on-aws.md]

### Requirements for `RelevanceToQuery`

The `RelevanceToQuery` judge assesses whether an application's response directly addresses the user's input. To use this judge, the [[MLflow Trace]] must have `inputs` and `outputs` present on the **root span** of the trace. If the root span does not contain these attributes, the judge cannot evaluate the relevance of the response. ^[answer-and-context-relevance-judges-databricks-on-aws.md]

### Requirements for `RetrievalRelevance`

The `RetrievalRelevance` judge evaluates whether each document returned by the application's retriever is relevant to the input request. The trace must contain at least one span whose `span_type` is set to `"RETRIEVER"`. This span typically captures the output of the retrieval step, including the documents that were returned. ^[answer-and-context-relevance-judges-databricks-on-aws.md]

If the trace is generated by a function decorated with `@mlflow.trace(span_type="RETRIEVER")`, the judge automatically inspects the documents returned by that function. For traces obtained via `mlflow.get_trace()`, the span with `span_type="RETRIEVER"` must exist for `retrieval_relevance()` to work. ^[answer-and-context-relevance-judges-databricks-on-aws.md]

### General Guidance

- When running evaluation with `mlflow.genai.evaluate()`, the user provides a `predict_fn` that generates the trace. The function should be instrumented with `@mlflow.trace` decorators as needed so that the required spans and attributes appear in the generated trace.
- When invoking a judge directly on a trace (e.g., `retrieval_relevance(trace=trace)`), the trace must already satisfy the requirements described above.
- If the trace does not meet the requirements, the judge may raise an error or return an incomplete result. ^[answer-and-context-relevance-judges-databricks-on-aws.md]
