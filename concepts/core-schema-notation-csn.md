# Core Schema Notation (CSN)

**Core Schema Notation (CSN)** is a notation used to describe data shares in a format that SAP Business Data Cloud (BDC) recipients can read. It is employed when sharing data assets from a Databricks workspace to an SAP BDC account via the [OpenSharing](/concepts/opensharing.md) protocol.

## Overview

After granting an SAP BDC recipient access to a share (a collection of tables or views), the share must be described in Core Schema Notation so that the SAP BDC users can consume the shared data. This description is generated by running a notebook that uses the [SAP BDC SDK](https://pypi.org/project/sap-bdc-connect-sdk/). ^[grant-sap-business-data-cloud-bdc-recipients-access-to-opensharing-data-shares-databricks-on-aws.md]

## Usage

CSN is not defined in the available source material beyond its role in the sharing workflow. The typical steps involving CSN are:

1. A share with data assets is created (any shared table must have a primary key or identity column).
2. The SAP BDC recipient is granted access to the share.
3. A notebook employing the SAP BDC SDK is executed to describe the share in Core Schema Notation, making the schema interpretable by SAP BDC systems.

The SAP BDC SDK takes care of translating the Databricks share metadata into the CSN format. ^[grant-sap-business-data-cloud-bdc-recipients-access-to-opensharing-data-shares-databricks-on-aws.md]
